# PK.md

A living document about how Phillip Klien operates as CEO of ClickBus. Published here so collaborators at ClickBus can reference it directly, and so other operators can borrow what's useful.

## What this is

A user manual for working with me. It exists for two reasons:

- **At ClickBus**, so people I work with — and the AI tools they use to help draft messages, prep meetings, or frame proposals — can calibrate without my time as a bottleneck.
- **Publicly**, because docs like this are better when people share them. If something here works as a template for your own version, take it.

It's not aspirational. It describes how I actually work, including the parts that are inconvenient.

## Who this is for

- People at ClickBus working with me directly or downstream of me.
- AI tools loaded with this as context to help draft, prep, or frame work that involves me.
- Other operators who want a starting point for their own "how I work" doc.

## Read it

→ [**PK.md**](./PK.md)

The doc has a built-in navigation map in §0 ("How to read this") that routes you to the right sections based on what you're trying to do — drafting a message, prepping a meeting, framing a proposal, pushing back, or just understanding context.

## How this evolves

This doc changes when I change. Suggestions make it more accurate, which is the whole point.

**At ClickBus**: see the "How to recommend updates to this doc" section at the bottom of PK.md for the Google Docs flow (suggesting mode, comments).

**On GitHub**:
- Spotted something outdated or unclear? Open an issue.
- Have a concrete edit? Open a pull request with a short rationale.
- Bigger rethink (new section, restructuring)? Open an issue first to discuss before drafting.

Either way: I read everything and decide what gets in.

## Versioning

PK.md has a version counter and "last updated" date at the top of the file. Git history is the real version trail; the counter is a human-readable signal that something meaningful changed.
